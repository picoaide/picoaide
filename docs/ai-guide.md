# AI 操作指引 -- PicoAide Web 管理面板文档系统

## 用途

本文档系统完整描述了 PicoAide Web 管理面板的所有操作功能、后端 API 设计、前端组件实现方案以及自动化测试场景。AI 在参与 PicoAide 项目开发时，应以此文档系统作为首要参考，理解功能逻辑后再进行代码修改或测试编写。

## 前置信息 -- 测试服务器

在阅读和验证文档内容时，可使用以下测试服务器环境：

| 项目 | 值 |
|------|-----|
| 服务器 IP | `10.88.7.22` |
| 完整地址（SERVER_URL） | `http://10.88.7.22` |
| 超管账号 | `admin` |
| 密码 | `admin123` |
| 服务端口 | 80（HTTP） |
| LDAP | 已预先配置好，可直接用于测试 LDAP 认证、组同步等功能 |

> **修改指引**：其他文档中的 `{SERVER_URL}` 占位符统一替换为实际服务器地址（如 `http://10.88.7.22`）。只需修改本文件中的服务器地址，然后全局搜索 `{SERVER_URL}` 替换即可。

## 文档阅读顺序

AI 应按以下顺序阅读文档，以建立从整体到局部的完整认知：

1. 先阅读本文（ai-guide.md），了解文档系统全貌
2. 阅读 `auth-source-guide.md`，理解认证源注册表模式和 LDAP/OIDC 实现
3. 阅读 `picoclaw-adapter-plan.md`，理解 Picoclaw 配置适配器体系
4. 阅读 `web-ui/` 目录下的 API 文档和页面组件文档（按具体任务需求）
5. 阅读 `mcp-testing/` 目录下的测试场景和模板（按具体任务需求）

## MCP Browser 工具说明

MCP Browser 是 AI 通过浏览器交互操作 PicoAide Web 管理面板的工具。其使用方式如下：

- AI 可通过 MCP Browser 在测试服务器上模拟用户操作，验证功能是否正常
- 操作包括：登录、用户管理、容器管理、镜像管理、技能管理、配置下发等
- 每次操作前应使用 `take_snapshot` 获取当前页面状态，再根据界面元素执行点击、填写表单等操作
- 结合 API 文档中的端点信息，AI 可以同时通过浏览器 UI 和 API 请求两种方式验证功能
