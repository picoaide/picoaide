# 测试场景 05：普通用户界面

## 前置条件
- PicoAide 服务运行在 {SERVER_URL}
- 当前认证模式为 local
- 存在至少一个本地普通用户（如 testuser，由超管在前序步骤创建）
- 超管账号: admin / admin123

## 测试用例

### TC-05-01: 超管创建本地用户
操作步骤：
1. 以超管 admin 登录
2. GET CSRF token: GET /api/csrf
3. POST /api/admin/users/create（form: username=testuser&password=test123&csrf_token=xxx）
4. 记录响应中返回的密码（系统可能自动生成）

预期结果：
- 返回 {"success":true,"username":"testuser"}
- 用户创建成功，容器自动启动
- GET /api/admin/users 显示 testuser 在列表中

### TC-05-02: 登出超管
操作步骤：
1. POST /api/logout
2. browser_navigate({ "url": "{SERVER_URL}/login" })

预期结果：
- 回到登录页
- session cookie 已清除

### TC-05-03: 普通用户登录
操作步骤：
1. 在登录页输入 testuser 的密码
2. 点击"登录"按钮

预期结果：
- 登录成功，跳转到 /manage
- 页面标题为"PicoAide 配置管理"
- 显示标签页：通讯渠道、文件管理、修改密码
- API 验证：GET /api/user/info 返回 {"role":"user","auth_mode":"local","unified_auth":false}

### TC-05-04: 通讯渠道标签
操作步骤：
1. 在 /manage 页面上点击"通讯渠道"标签

预期结果：
- 显示渠道配置区域
- 提示管理员尚未开放渠道或显示可用渠道列表
- 管理员关闭渠道时显示"管理员还没有开放可配置的通讯渠道"
- 有渠道时显示渠道列表和"保存并重启容器"按钮

### TC-05-05: 文件管理标签
操作步骤：
1. 点击"文件管理"标签
2. 查看文件列表

预期结果：
- 显示 .picoclaw/workspace/ 下的文件和目录
- 显示面包屑导航（如"工作区"）
- 每个文件显示大小，有"下载"和"删除"按钮
- 每个目录有"删除"按钮
- 有"新建目录"、"选择文件上传"按钮

### TC-05-06: 文件上传
操作步骤：
1. 在文件管理标签
2. 选择文件并上传到当前目录

预期结果：
- 文件上传成功
- 文件出现在目录列表中

### TC-05-07: 文件下载
操作步骤：
1. 点击某个文件的"下载"按钮

预期结果：
- 文件下载成功

### TC-05-08: 文件删除
操作步骤：
1. 点击某个文件的"删除"按钮

预期结果：
- 文件被删除
- 文件不再显示在列表中

### TC-05-09: 创建目录
操作步骤：
1. 点击"新建目录"按钮
2. 输入目录名
3. 确认

预期结果：
- 目录创建成功
- 新目录出现在文件列表中

### TC-05-10: 修改密码标签
操作步骤：
1. 点击"修改密码"标签

预期结果：
- 显示密码修改表单
- 字段：旧密码、新密码、确认新密码
- 新密码要求至少 6 个字符
- 有"修改密码"提交按钮

### TC-05-11: 修改密码成功
操作步骤：
1. 在修改密码表单中
2. 输入旧密码
3. 输入新密码（6 位以上）
4. 输入确认新密码
5. 点击"修改密码"

预期结果：
- 返回 {"success":true,"message":"密码修改成功"}
- 用新密码可以重新登录
- 用旧密码登录被拒绝

### TC-05-12: 修改密码失败（旧密码错误）
操作步骤：
1. 在修改密码表单中
2. 输入错误的旧密码
3. 输入新密码
4. 提交

预期结果：
- 返回 {"success":false,"error":"旧密码错误"}

### TC-05-13: 修改密码失败（新密码太短）
操作步骤：
1. 输入正确的旧密码
2. 输入少于 6 位的新密码
3. 提交

预期结果：
- 返回 400 错误
- 提示"新密码至少 6 个字符"

### TC-05-14: 普通用户获取 MCP Token
操作步骤：
1. 以普通用户 testuser 登录
2. GET /api/mcp/token

预期结果：
- 返回 {"success":true,"token":"testuser:xxx..."}
- token 格式为 "用户名:64位hex"

### TC-05-15: 超管不能获取 MCP Token
操作步骤：
1. 以超管 admin 登录
2. GET /api/mcp/token

预期结果：
- 返回 403 "超管用户不允许登录插件"

### TC-05-16: 用户信息 API
操作步骤：
1. 以普通用户登录
2. GET /api/user/info

预期结果：
- 返回 {"role":"user","auth_mode":"local","unified_auth":false}
- 不返回 superadmin 或超管相关字段

### TC-05-17: 初始化状态 API
操作步骤：
1. 以普通用户登录
2. GET /api/user/init-status

预期结果：
- 返回 {"ready":true/false,"status":"running/...","image_ready":true,"has_config":true}

### TC-05-18: 普通用户不能访问超管端点
操作步骤：
1. 以普通用户登录
2. GET /api/admin/users

预期结果：
- 返回 403 "仅超级管理员可访问"
- 超管端点的写入操作（创建/删除用户等）同样被拒绝

### TC-05-19: 统一认证模式下修改密码被拒
前置条件：
- 当前模式为 ldap 或 oidc
- 用户为 LDAP/OIDC 同步的外部用户

操作步骤：
1. 以外部用户登录
2. POST /api/user/password（old_password=xxx, new_password=yyy, csrf_token=xxx）

预期结果：
- 返回 403 "非本地用户不支持修改密码，请联系管理员在公司认证中心修改"

## 清理
- 如测试中创建了多余文件/目录，在文件管理中删除
- 将密码恢复为测试前状态（如修改过）
- 留下测试用户 testuser 用于后续测试
