# 测试场景 06：团队空间（共享文件夹）

## 前置条件
- PicoAide 服务运行在 {SERVER_URL}
- MCP Browser 工具可用
- 超管账号: admin / admin123
- 已创建至少 2 个用户组（如「研发部」「产品部」），各含至少 1 个组成员
- 已创建至少 3 个普通用户（如 user1、user2、user3）

## 测试用例

### TC-06-01: 超管访问团队空间页面
**涉及功能**：管理后台 - 团队空间页

操作步骤：
1. browser_navigate({ "url": "{SERVER_URL}/login" })
2. browser_type({ "selector": "#username", "text": "admin" })
3. browser_type({ "selector": "#password", "text": "admin123" })
4. browser_click({ "selector": "#login-btn" })
5. browser_wait({ "selector": ".sidebar-nav", "timeout": 5000 })
6. browser_click({ "selector": "a[data-section='teamspace']" })
7. browser_screenshot({})

预期结果：
- 侧边栏出现「团队空间」菜单项
- 页面标题包含「团队空间」
- 左侧为空列表（暂无共享文件夹）
- 右侧显示「请选择一个共享文件夹」

---

### TC-06-02: 创建共享文件夹（关联组）
**涉及功能**：团队空间 - 创建共享文件夹

操作步骤：
1. 在团队空间页面，点击「新建共享文件夹」按钮
2. browser_type({ "selector": "#sf-name", "text": "项目文档" })
3. browser_type({ "selector": "#sf-description", "text": "项目相关共享文档" })
4. browser_click({ "selector": "#sf-is-public" })  // 确保不勾选
5. 在「关联用户组」下拉列表中勾选「研发部」
6. browser_click({ "selector": "#sf-save-btn" })

预期结果：
- 显示「创建成功」提示
- 左侧列表出现「项目文档」
- API 验证：GET /api/admin/shared-folders 返回包含刚创建的共享文件夹

---

### TC-06-03: 创建公共共享文件夹
**涉及功能**：团队空间 - 创建公共共享文件夹

操作步骤：
1. 点击「新建共享文件夹」按钮
2. browser_type({ "selector": "#sf-name", "text": "公司公告" })
3. browser_type({ "selector": "#sf-description", "text": "公司全员可见" })
4. browser_click({ "selector": "#sf-is-public" })  // 勾选公共
5. 不关联任何用户组
6. browser_click({ "selector": "#sf-save-btn" })

预期结果：
- 显示「创建成功」提示
- 左侧列表出现「公司公告」，标记为「公共」
- 主机目录 work_dir/shared/公司公告/ 已创建

---

### TC-06-04: 查看共享文件夹成员列表
**涉及功能**：团队空间 - 成员列表

操作步骤：
1. 点击左侧「项目文档」
2. browser_wait({ "selector": "#sf-member-list", "timeout": 3000 })
3. browser_get_content({ "selector": "#sf-member-list" })

预期结果：
- 右侧详情显示共享文件夹信息（名称、描述、类型、关联组）
- 成员列表中显示研发部的所有用户（含子组成员）
- 每个成员显示挂载状态（未挂载 / 已挂载）

---

### TC-06-05: 测试单个用户挂载状态
**涉及功能**：团队空间 - 测试挂载

操作步骤：
1. 选中「项目文档」共享文件夹
2. 在成员列表中，找到某个用户，点击其行上的「测试挂载」按钮
3. browser_wait({ "selector": ".msg-info", "timeout": 5000 })

预期结果：
- 显示测试结果：如「用户 user1 已挂载」或「用户 user1 未挂载」
- 成员列表中的该用户状态更新为 ✓ 或 ✗
- 如果用户没有运行的容器，提示「容器未运行」

---

### TC-06-06: 一键挂载所有用户
**涉及功能**：团队空间 - 一键挂载

操作步骤：
1. 选中「项目文档」共享文件夹
2. 点击「一键挂载」按钮
3. browser_wait({ "selector": ".task-progress", "timeout": 10000 })

预期结果：
- 显示异步任务进度（如「处理中: 3/5」、「完成: 3，失败: 0」）
- 任务完成后，成员列表中所有运行容器的用户状态变为「已挂载」
- API 验证：GET /api/admin/task/status 返回任务完成状态

---

### TC-06-07: 组成员变更自动触发挂载变更
**涉及功能**：团队空间 - 组成员添加触发重启

操作步骤：
1. 在用户组页面，向「研发部」添加一个新用户 user3
2. 返回团队空间页面，选中「项目文档」
3. 查看成员列表

预期结果：
- user3 出现在「项目文档」的成员列表中
- user3 的容器已自动重启并挂载共享文件夹（如果有运行容器）
- 如果容器未运行，显示「未挂载」

---

### TC-06-08: 修改共享文件夹名称并验证容器重启
**涉及功能**：团队空间 - 改名

操作步骤：
1. 选中「项目文档」共享文件夹
2. 点击「编辑」按钮
3. 修改名称为「项目文档-v2」
4. browser_click({ "selector": "#sf-save-btn" })
5. browser_wait({ "selector": ".msg-success", "timeout": 5000 })

预期结果：
- 显示「修改成功，正在重启容器（N 个用户）」
- 左侧列表名称变为「项目文档-v2」
- 主机目录从 shared/项目文档/ 变更为 shared/项目文档-v2/
- 重启的用户在容器内新的路径可见共享文件

---

### TC-06-09: 公共共享所有用户可见
**涉及功能**：团队空间 - 公共共享（用户端）

操作步骤：
1. 登出超管账号
2. 用普通用户（user1）登录
3. 访问 /manage 页面
4. 点击「团队空间」tab

预期结果：
- 用户端出现「团队空间」tab
- 显示「公司公告」公共共享文件夹
- 显示「项目文档-v2」（如果 user1 是研发部成员）
- 每个共享文件夹显示名称、路径（workspace/share/xxx/）、类型

---

### TC-06-10: 删除共享文件夹并验证归档和重启
**涉及功能**：团队空间 - 删除

操作步骤：
1. 登出普通用户，用超管 admin 登录
2. 进入团队空间页面
3. 选中「项目文档-v2」共享文件夹
4. 点击「删除」按钮
5. 确认删除弹窗
6. browser_wait({ "selector": ".msg-success", "timeout": 5000 })

预期结果：
- 显示「删除成功」提示
- 主机目录移动到 archive/shared_项目文档-v2_<时间戳>/
- 所有已挂载用户的容器立即重启（卸载共享文件夹）
- 团队成员在容器内无法再访问该目录
- API 验证：GET /api/admin/shared-folders 不再返回该记录

---

### TC-06-11: 组删除时自动清理共享文件夹关联
**涉及功能**：团队空间 - 组删除级联

操作步骤：
1. 创建一个新共享文件夹，关联「产品部」
2. 在用户组页面删除「产品部」
3. 返回团队空间页面

预期结果：
- 该共享文件夹不再关联任何组（如果无其他关联组且非公共，标记为「孤立」）
- 原产品部成员的容器已重启（从共享中移除）
- 如果文件夹无组且非公共 → UI 显示红色警告「无关联组，无人可访问」

---

### TC-06-12: 用户删除时清理挂载记录
**涉及功能**：团队空间 - 用户删除级联

操作步骤：
1. 确保某个用户（如 user2）在某个共享文件夹的成员列表中
2. 在用户管理页删除 user2
3. 返回团队空间页面，查看该共享文件夹的成员列表

预期结果：
- 成员列表中不再包含 user2
- 数据库 shared_folder_mounts 表中无 user2 的记录

---

### TC-06-13: 名称冲突被拒绝
**涉及功能**：团队空间 - 唯一性验证

操作步骤：
1. 尝试创建名称为「公司公告」的共享文件夹（名称已存在）
2. 点击保存

预期结果：
- 显示错误提示「名称已被占用，请使用其他名称」
- 列表没有新增条目

---

### TC-06-14: 超管不可见
**涉及功能**：团队空间 - 超管不参与共享

操作步骤：
1. 用超管账号创建一个公共共享文件夹
2. 查看成员列表

预期结果：
- 超管账号不显示在成员列表中
- 超管的工作端没有团队空间 tab

---

### TC-06-15: 容器启动时自动挂载
**涉及功能**：团队空间 - 容器创建自动挂载

操作步骤：
1. 确保某普通用户有资格访问一个共享文件夹
2. 如果容器未运行，启动该用户的容器
3. 执行 Docker exec 检查容器内共享目录

预期结果：
- 容器创建/重启时自动挂载所有可访问的共享文件夹
- `docker exec <container> ls /root/.picoclaw/workspace/share/` 包含共享文件夹名称
- 目录内容与主机 shared/<名称>/ 一致

---

### TC-06-16: 异步挂载任务状态查看
**涉及功能**：团队空间 - 异步任务进度

操作步骤：
1. 选中一个关联了多个用户的共享文件夹
2. 点击「一键挂载」按钮
3. 观察进度显示
4. browser_execute({ "script": "return fetch('{SERVER_URL}/api/admin/task/status').then(r=>r.json())" })

预期结果：
- 前度显示实时更新（X/N 完成）
- API 返回任务状态：running/finished，done/failed 计数
- 完成后显示总结：「完成：N 成功，M 失败」
