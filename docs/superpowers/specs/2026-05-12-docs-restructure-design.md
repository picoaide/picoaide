# PicoAide 文档体系重构设计

## 概述

对 PicoAide 官网（Hugo 静态站点）的文档结构进行全面重构，从原有的 8 篇文档扩展至 13 篇，新
增管理员手册和用户指南系列文档，按角色分层组织。

## 设计目标

1. **按角色分层**：管理员线和用户线文档清晰分离，各自覆盖完整使用流程
2. **内容深化**：每篇文档的篇幅和深度显著增加，覆盖实操场景和最佳实践
3. **数量扩展**：从 8 篇增至 13 篇，新增 6 篇，重写 5 篇，保留 2 篇
4. **设计哲学融入**：在架构文档中系统阐述"AI 的力量应当被释放，但数据的边界必须被守护"的核
   心理念

## 各文档内容大纲

### 1. 快速入门 (quick-start.md)
面向所有新用户，从 0 到可访问后台。涵盖环境要求、安装方式、初始化向导、systemd 服务、验证安装。

### 2. 架构设计 (architecture.md)
核心设计哲学展开，系统总览图，五大设计决策（控制平面、容器隔离、认证可插拔、MCP 中继、配置分层下发）。

### 3. 安装与部署 (install-deploy.md)
深度安装指南：环境规划、安装方式详解、初始化最佳实践、服务管理、升级流程、数据迁移、多实例注意事项。

### 4. 管理后台操作指南 (admin-guide.md)
超管 `/admin/*` 完整操作手册：用户管理、容器管理、镜像管理、技能管理、用户组管理、团队空间、渠道策略、全局设置。

### 5. 认证与安全配置 (auth-security.md)
三种认证模式详解：local/LDAP/OIDC 配置、白名单、超管账户、切换认证模式的数据清理规则。

### 6. 系统配置参考 (configuration.md)
配置字段完整参考：Web、镜像、技能仓库、Adapter 包、安全权限、CLI 命令。

### 7. 故障排查 (troubleshooting.md)
按类型组织的故障诊断：容器问题、认证问题、代理连接、配置问题、扩展/客户端问题、日志与调试。

### 8. Web 面板操作指南 (web-panel.md)
普通用户 `/manage` 操作手册：频道管理、文件管理、团队空间、密码修改、MCP Token。

### 9. 浏览器扩展 (browser-extension.md)
Chrome 扩展安装与使用：授权流程、Cookie 同步、Browser MCP 工具、常见场景。

### 10. 桌面客户端 (desktop-client.md)
桌面客户端安装使用：连接流程、权限组、文件白名单、Computer MCP 工具、安全建议。

### 11. MCP 与 AI 集成 (mcp-integration.md)
MCP 协议说明、Token 获取、工具配置、验证可用性、browser/computer 工具组合最佳实践。

### 12. API 参考 (api.md)
完整 API 端点列表：保留现有内容。

### 13. 常见问题 (faq.md)
高频问题汇总：保留现有内容并同步更新。
